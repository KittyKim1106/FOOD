# 犹豫就会饿死

一个用来解决“今天吃什么”的趣味静态前端项目。  
项目主入口是 [`load.html`](/d:/XiangMu/前端/chishenme/html/load.html)，通过单文件方式整合了启动页、意向选择、滑动决策、结果页、失败页、设置页和历史页。

## 项目特点

- 纯前端静态页面，无需安装依赖
- 主应用基于 React 18 + React Router DOM 运行
- 使用 Tailwind CSS CDN 快速完成界面样式
- 页面风格统一，适合做交互原型、课程作业或创意展示
- 同目录下保留了若干独立 HTML 页面，方便拆页演示

## 主要功能

- 启动画面：进入应用后先展示品牌与开始按钮
- 意向调查：先勾选想吃的食物类型，如海鲜、蔬菜、面、米饭等
- 滑动决策：通过卡片式交互快速筛选候选食物
- 结果钦定：给出最终推荐结果
- 全军覆没：没有合适选项时展示兜底页面
- 排雷设置：排除不想吃的口味、食材或偏好
- 历史记录：查看此前的选择结果

## 技术栈

- React 18 UMD
- React DOM 18 UMD
- React Router DOM 6
- Tailwind CSS CDN
- Babel Standalone
- Google Fonts `Space Grotesk`
- Material Symbols

## 页面说明

### `load.html`

完整主应用，内部包含以下路由页面：

- `/`：启动页
- `/home`：意向调查页
- `/swipe`：滑动决策页
- `/result`：结果页
- `/failure`：失败页
- `/settings`：设置页
- `/history`：历史页

### 其他独立 HTML

目录下的中文命名页面可以理解为单独拆出来的静态演示稿，适合单页预览、讲解或对照设计稿使用。

## 如何运行

### 方式一：直接打开

直接用浏览器打开 [`load.html`](/d:/XiangMu/前端/chishenme/html/load.html) 即可。

### 方式二：本地静态服务器

如果你想避免某些浏览器对本地文件的限制，可以在当前目录启动一个静态服务器，例如：

```powershell
python -m http.server 8000
```

然后访问 `http://localhost:8000/load.html`。

## 使用说明

1. 打开首页后点击“立即开启”
2. 在意向调查页选择想吃的品类
3. 进入滑动决策页继续筛选
4. 查看系统给出的推荐结果
5. 如有需要，可进入设置页排除不喜欢的内容
6. 在历史页查看过往记录

## 设计说明

- 主色为橙色，强调食欲和行动感
- 采用卡片式布局与较强的视觉层级
- 同时适配浅色与深色背景
- 更偏移动端体验，也可以在桌面浏览器中演示

## 注意事项

- 项目虽然不需要安装 npm 依赖，但默认依赖外部 CDN 资源
- 首次打开时需要联网加载 React、Tailwind、字体和图标
- 如果要真正离线使用，需要把这些 CDN 资源改为本地文件
